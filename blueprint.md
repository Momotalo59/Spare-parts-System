
# Blueprint: ระบบคลังพัสดุและเครื่องมือ

## 1. ภาพรวมโปรเจกต์ (Project Overview)

แอปพลิเคชันนี้คือระบบบริหารจัดการคลังพัสดุและเครื่องมือสำหรับแผนกช่างเทคนิคของโรงพยาบาลโอเวอร์บรุ๊ค ผู้ใช้สามารถดูรายการพัสดุ, เบิก/จ่าย/คืนพัสดุผ่านการสแกน QR Code, และดูประวัติการใช้งานได้ ผู้ดูแลระบบ (Admin) จะมีสิทธิ์ในการเพิ่ม, ลบ, หรือแก้ไขรายการพัสดุ

---

## 2. โครงสร้างและดีไซน์ (Original UI & Structure)

โปรเจกต์มีโครงสร้างและดีไซน์ที่เป็นเอกลักษณ์ ดังนี้:

### 2.1 Header (ส่วนหัว)
- **ตำแหน่ง:** ด้านบนสุดของทุกหน้า
- **องค์ประกอบ:**
    - **ซ้าย:** โลโก้โรงพยาบาลโอเวอร์บรุ๊ค พร้อมชื่อระบบ "คลังพัสดุและเครื่องมือ" และคำอธิบาย
    - **ขวา:** ข้อความต้อนรับ "ยินดีต้อนรับ, [ชื่อผู้ใช้]" พร้อมปุ่ม Logout

### 2.2 Footer (ส่วนท้าย)
- **ประเภท:** Bottom Navigation Bar แบบ固定 (Fixed) อยู่ด้านล่างสุดของจอ
- **องค์ประกอบ (เมนูหลัก):**
    1.  **หน้าหลัก (Home):** ไอคอนรูปบ้าน, ลิงก์ไปที่ `/`
    2.  **สแกน (Scan):** ไอคอนรูป QR Code, ลิงก์ไปที่ `/scan`
    3.  **ประวัติ (History):** ไอคอนรูปนาฬิกา, ลิงก์ไปที่ `/history`

### 2.3 Dashboard (หน้าหลัก)
- แสดงรายการพัสดุทั้งหมด โดยแบ่งตามหมวดหมู่ผ่าน Tabs:
    - รายการอะไหล่
    - รายการเครื่องมือ
    - อุปกรณ์เซฟตี้
- มีช่องสำหรับค้นหา (Search Bar)
- สำหรับ Admin: จะมีแผงควบคุม (Admin Panel) สำหรับ "เพิ่มรายการ" และดู "ประวัติการใช้งาน"

## 3. แผนการกู้คืน (Restoration Plan) - ครั้งล่าสุด

**ปัญหาปัจจุบัน:**
1.  **UI Regression:** Header และ Footer ถูกเขียนทับด้วยเวอร์ชันที่ไม่ถูกต้องระหว่างการแก้ไขข้อผิดพลาด
2.  **Data Loss:** ข้อมูลพัสดุไม่แสดงผลในหน้า Dashboard (ขึ้นว่า "ไม่มีรายการในหมวดนี้")
3.  **Auth Data Error:** Header แสดงข้อความ "ยินดีต้อนรับ, undefined"

**ขั้นตอนการแก้ไข:**

1.  **[ดำเนินการ] กู้คืน `App.jsx` (Layout หลัก):**
    - นำ `Header` และ `AppNav` (Footer) ที่ถูกต้องกลับเข้ามาเป็น Layout หลัก
    - จัดการการดึงข้อมูล `userData` ให้ถูกต้อง เพื่อส่ง `displayName` ไปให้ `Header`
    - แก้ไขให้ `Routes` แสดงผลอยู่ตรงกลางระหว่าง Header และ Footer

2.  **[ดำเนินการ] กู้คืน `Header.jsx`:**
    - ตรวจสอบและยืนยันว่าโค้ดใน `src/components/Header.jsx` ตรงตามดีไซน์ดั้งเดิม (มีโลโก้และชื่อระบบ)

3.  **[ดำเนินการ] กู้คืน `AppNav.jsx` (Footer):**
    - เขียนโค้ดสำหรับ Bottom Navigation Bar ขึ้นมาใหม่ให้ตรงตามดีไซน์ดั้งเดิม (หน้าหลัก, สแกน, ประวัติ)

4.  **[ดำเนินการ] กู้คืนข้อมูลใน `Dashboard.jsx`:**
    - ตรวจสอบและแก้ไข Logic การดึงข้อมูลจาก Firestore (`onSnapshot`) เพื่อให้มั่นใจว่าข้อมูลพัสดุถูกดึงมาและแสดงผลอย่างถูกต้อง
